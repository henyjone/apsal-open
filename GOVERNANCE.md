# Governance

APSAL Open uses maintainer-reviewed governance. The canonical schemas, compiler behavior, official namespace, and release keys are protected branches. Community DNA may use contributor-owned namespaces and is accepted when schema, rights, lineage, and QA checks pass.

Published versions are immutable. Generation-intent changes require a new semantic version. Corrections that do not change intent use a patch version. Visual quality remains a human review decision.

Recommendation vocabulary and Extension Pack schema changes are reviewed as Registry-layer RFCs. A recommendation score is advisory and explainable, not an official aesthetic ranking. Personal usage memory remains local and is never accepted as community evidence without an explicit creator submission.
