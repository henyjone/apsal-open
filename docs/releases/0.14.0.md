# APSAL Studio 0.14.0

APSAL Studio 0.14 adds a visual summary of the full five-layer design process without bringing back the distracting image-based DNA cards. It also makes real reference handling explicit inside every newly exported theme Skill.

## Install or upgrade

```bash
codex plugin marketplace add henyjone/apsal-open --ref v0.14.0
codex plugin add apsal-studio@apsal-open
```

Restart Codex or open a new task. `codex plugin list` should show `apsal-studio@apsal-open` at `0.14.0`.

## What creators see

- DNA recommendations remain complete text-only cards.
- Above the element cards, five 4:3 semantic thumbnails summarize Direction, Worldbuilding, Narrative, Image and Delivery.
- Chinese sessions use Chinese thumbnails and cards; English sessions use English.
- Confirmed, current and pending stages are visually distinct.
- The text cards still contain the complete proposal, rationale, options, values, observable effects, locks and QA. A thumbnail never replaces them.

## What the Skill contains

Every new Prompt/Skill ZIP includes ten localized SVG stage summaries and `references/preview_manifest.json`. These assets are deterministic, rights-clear and marked `generation_input: false`.

When real references exist, `references/reference_manifest.json` designates exactly one core visual anchor and includes the sanitized image bytes under `assets/references/`. The anchor remains limited by its declared uses, forbidden uses and applicable Jobs. With no real image, the package says `not_bound`; it never substitutes a stage thumbnail.

## Compatibility

Protocol and Semantic Contract remain 0.3.0. Session schema remains 0.7.0. Reference manifest becomes 0.6.0 and the Prompt/Skill manifest becomes 0.10.0. Existing frozen themes, DNA assets and older packages remain readable. Stage previews do not change canonical theme or DNA digests.

Static tests validate structure, checksums and separation of preview/reference assets. Generated images still require model and human visual QA.
