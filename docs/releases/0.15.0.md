# APSAL Studio plugin 0.15.0

APSAL 0.15 introduces one versioned Python project kernel shared by the Codex plugin and APSAL Studio Desktop 0.2.0.

For the complete architecture, data model, API contract, implementation order, cutover procedure, and acceptance matrix, follow the [0.15.0 Chinese upgrade specification](../UPGRADE_GUIDE_0.15.0.zh-CN.md).

## Install

```bash
codex plugin marketplace add henyjone/apsal-open --ref v0.15.0
```

After restarting Codex, `codex plugin list` should show `apsal-studio@apsal-open` at `0.15.0`.

## What changed

- `.apsal/project.json` now records project ID, protocol version, engine version, active session, and monotonic revision.
- Semantic writes use project locking, `expectedRevision`, idempotent `operationId`, atomic files, and crash recovery.
- An `operationId` is permanently bound to one normalized intent; reusing it for a different change is rejected.
- The same domain dispatcher runs in-process for MCP and over stdio JSON-RPC for the Electron sidecar.
- Seven `apsal_frontend_*` tools discover the optional loopback-only Studio bridge, preview changes, confirm/reject, undo, and focus stable elements.
- `start_design_session` records the creator's one-time `studio` or `headless` frontend choice. Studio is launched and authenticated by the plugin only after opt-in; an independently opened app is never treated as consent to link.
- Studio view state is isolated in `.apsal/studio/view.json` and excluded from theme digests and final packages.
- Protocol/Engine mismatches are readable but cannot be upgraded or written in place; start a new 0.15 project instead.
- Final Prompt/Skill ZIPs include a deterministic in-package SHA-256 ledger in addition to the outer archive checksum.
- Golden contract tests verify direct and sidecar paths return identical snapshots and deterministic ZIP bytes/SHA-256.

Codex remains fully functional when Studio is unconfigured, closed, offline, or declined. APSAL Studio's Codex linkage is off by default and has no manual frontend switch; Codex asks before launching it.
