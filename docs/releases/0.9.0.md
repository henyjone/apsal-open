# APSAL Studio 0.9.0

APSAL Studio can now open an old APSAL run ZIP and take responsibility for the whole experience. Creators no longer need to extract it, inspect `run.json`, fix paths, or install an image API runner.

## Install

```bash
codex plugin marketplace add henyjone/apsal-open --ref v0.9.0
codex plugin add apsal-studio@apsal-open
```

Restart Codex or open a new task.

For the full creator workflow—including new-theme authoring, all five layers, DNA memory, reference roles, nine-image continuation, Prompt/Skill installation and troubleshooting—read the [complete usage guide](../USAGE_GUIDE.md) or [中文完整使用手册](../USAGE_GUIDE.zh-CN.md).

## Use an existing package

Attach the APSAL ZIP and say:

> 打开这个 APSAL 包并生成第一张图。

Studio automatically:

1. safely reads the one `run.json`;
2. restores every shot Prompt;
3. ignores obsolete API execution fields while preserving them as lineage;
4. finds omitted references by SHA-256 in the ZIP or private Vault;
5. creates a new private Codex Prompt/Skill ZIP;
6. hands SHOT_01 to Codex built-in image generation.

If a reference is genuinely unavailable, Studio asks only for that image. Once reattached, its SHA-256 must match before generation continues.

## CLI

```bash
python3 plugins/apsal-studio/scripts/apsal.py import-run path/to/legacy-run.zip
python3 plugins/apsal-studio/scripts/apsal.py run-bind-reference RUN-ID REFERENCE-ID path/to/reference.png
python3 plugins/apsal-studio/scripts/apsal.py run-next RUN-ID
```

## Safety and output

- ZIP traversal, symlinks, duplicates and decompression abuse are rejected.
- Imported packages are always private-only.
- No image API, provider endpoint or API Key is used.
- Each Codex turn generates one finished photograph, never code or a programming interface.
- 9:16 and 2160×3840 may be preserved as creative delivery requests, not returned-pixel guarantees.
- Model and human visual QA remain separate.
