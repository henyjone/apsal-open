# APSAL Studio 0.7.0

APSAL Studio 0.7.0 makes the complete APSAL method visible to creators. The former Character / World / Scene / Photo confirmation is replaced for new sessions by five text-card layers that cover all thirteen existing protocol roles.

Protocol and Semantic Contract remain 0.3.0. The official DNA catalog and existing theme generation intent are unchanged.

## Install

```bash
codex plugin marketplace add henyjone/apsal-open --ref v0.7.0
codex plugin add apsal-studio@apsal-open
```

Restart Codex or open a new task after installation.

## New creator flow

1. Direction — Content and Emotion.
2. Worldbuilding — Subject, World and Look.
3. Narrative — Event and Sequence.
4. Image — Camera, Light, Style and Color/Post.
5. Delivery — Job and Quality Control.

The element cards are text-only. They display purpose, source, structured values, observable effects, invariants and QA expectations. Creators can accept one layer or revise a single element in natural language.

The seven Registry categories remain Character, Environment, Composition, Shot, Style, Lighting and QA. They are recommended in the relevant layer, can be saved to My DNA only after explicit permission, and can still be exported as independent Extension Packs.

## What is now explicit

- overall proposition, primary and optional secondary moods, undertone, valence, arousal, expression, energy, tension and emotional arc;
- subject identity, space, time, materials, wardrobe, grooming, props and ownership;
- event consequences, shot order, rhythm and nine distinct narrative functions;
- viewpoint, coverage, framing, optics, composition and light source/direction/quality/contrast;
- palette, temperature, saturation, contrast curve, grain, sharpness, dynamic range and skin-tone policy;
- one-Job-one-image delivery, rejection conditions, model visual QA and separate human visual QA.

Finalization fails until all thirteen roles are confirmed. A changed upstream layer invalidates every affected downstream layer and compiled Prompt.

## Compatibility

- Existing Studio 0.4–0.6 JSON, YAML, themes, Registry assets, Skills and runs remain readable.
- Legacy four-stage sessions continue through their legacy state path and are not falsely labelled as thirteen confirmed decisions.
- Studio/Engine version: 0.7.0.
- Protocol version: 0.3.0.
- Semantic Contract version: 0.3.0.
