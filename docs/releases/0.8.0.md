# APSAL Studio 0.8.0

APSAL Studio now completes the whole creator journey inside Codex: it designs the world, packages every Prompt and permitted reference, and hands one frozen Job at a time to Codex built-in image generation. It no longer calls an image API or asks for an image API key.

## Install

```bash
codex plugin marketplace add henyjone/apsal-open --ref v0.8.0
codex plugin add apsal-studio@apsal-open
```

Restart Codex or open a new task, then say:

> 创建一套九张东方极简窗边真人摄影主题。

## Creator flow

1. Confirm Direction, Worldbuilding, Narrative, Image and Delivery through compact text cards.
2. Review all thirteen roles, nine shots, live-action contract, references and rights.
3. Finalization automatically creates the theme artifacts and a documented Codex Prompt/Skill ZIP.
4. Choose “现在用 Codex 生成” once. Codex generates SHOT_01 with its built-in image-generation capability.
5. Inspect the image and say “继续”. Codex advances to the next unfinished Job. Each turn produces only one independent image.

If you only need the Prompts, open `PROMPT_GUIDE.md` in the exported ZIP and copy any `prompts/SHOT_XX.full.txt`. No JSON/YAML editing and no image API configuration are required.

## Prompt package

The package contains 27 human-readable Prompt files: positive, negative and combined Prompt variants for each of nine Jobs. It also contains the theme, design context, QA checklist, live-action Rendering Contract, reference usage/rights manifest, actual reference images when allowed, file checksums and an offline validator.

Run this after extracting the Skill:

```bash
python3 scripts/validate_prompt_pack.py --list
```

The command verifies the Prompt and reference checksums and lists all Jobs. It never generates an image or makes a network request.

## Output contract

- Generation surface: Codex built-in image generation.
- Direct image API calls: disabled.
- API key: not required.
- One Job per image and one image-generation call per Codex turn.
- Requested output: 9:16, high quality, creative delivery target 2160×3840.
- Returned format, model and dimensions: recorded only when Codex reports them; otherwise `not_reported`.
- Native/returned 4K guarantee: no.

Model visual QA and human visual QA remain separate. Passing a schema or Prompt check never proves photographic quality.
