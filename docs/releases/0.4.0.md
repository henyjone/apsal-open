# APSAL Studio 0.4.0

APSAL Studio now starts with one natural-language idea and guides the creator through clickable Character, World, Scene, and Photo DNA cards. The creator confirms a nine-shot overview, then generates nine independent images, saves the Prompts, or exports a Skill. JSON and YAML remain local background assets.

## Install in three steps

```bash
codex plugin marketplace add henyjone/apsal-open --ref v0.4.0
codex plugin add apsal-studio@apsal-open
```

Restart Codex or open a new task, then say:

> 创建一套九张东方极简窗边人像主题。

- APSAL Open Protocol: 0.3.0 (unchanged)
- Semantic Contract: 0.3.0 (unchanged)

- APSAL Studio plugin: 0.4.0
- Existing Protocol 0.2 JSON, Protocol 0.3 YAML/JSON and older exported Skills remain readable.

## Privacy and QA

Core authoring, Registry resolution, validation, compilation and Skill packaging are local and require no account or hosted API. Image generation is optional and requires explicit confirmation. Private character references remain in the local content-addressed Vault and are excluded from releases and Skills.

The seven bundled preview images are original abstract semantic cards. They help explain a DNA choice; they are not evidence that generated photography has passed visual QA.
