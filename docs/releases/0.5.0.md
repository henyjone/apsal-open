# APSAL Studio 0.5.0

APSAL Studio 0.5 closes three gaps in theme execution: the image model now receives the actual bound references, “real person” is an explicit live-action photography contract rather than a weak adjective, and a nine-shot request becomes nine resumable native-4K Jobs instead of an unsupported single batch.

## Install

```bash
codex plugin marketplace add henyjone/apsal-open --ref v0.5.0
codex plugin add apsal-studio@apsal-open
```

Restart Codex or open a new task. Then ask:

> Create a nine-shot 9:16 live-action portrait theme. Confirm each reference's allowed use before generating.

## Release assets

- `apsal-studio-codex-plugin-v0.5.0.zip` — self-contained Registry, engine, local MCP, schemas, Skill, reference packager, and Skill executor.
- `apsal-studio-codex-plugin-v0.5.0.zip.sha256` — SHA-256 checksum.

```bash
shasum -a 256 -c apsal-studio-codex-plugin-v0.5.0.zip.sha256
```

## What changes for creators

- New themes default to nine independent 9:16, 2160×3840 high-quality PNG Jobs.
- Every supplied image is bound as style, world, prop, wardrobe, composition, or identity, with allowed/forbidden uses and rights metadata.
- A local private Skill includes sanitized reference copies so the provider sees them. Unverified media can never be published by the packager.
- The live-action contract requires real adult-human photographic rendering while sets and props may remain handmade, crayon-like, painted, or theatrical.
- One confirmation starts the run; the adapter makes nine sequential `n: 1` requests, retries failures twice, and resumes without overwriting successful shots.
- `SHOT_01` may anchor only the fictional identity for later frames. Pose, camera, background, wardrobe, action, and composition do not carry over.

## Native 4K boundary

Guaranteed execution requires the optional OpenAI Image API adapter, `gpt-image-2`, and a user-supplied `OPENAI_API_KEY`. The key is read from the environment only. Every output must parse as exactly 2160×3840 before APSAL records success.

Codex ImageGen is not a silent native-4K fallback because its local tool surface does not expose the required size contract. Without a key, export Prompts/Skill or explicitly accept “dimensions not guaranteed.” GPT Image 2 supports 2160×3840, but output above 2K is experimental; see the [official image generation guide](https://developers.openai.com/api/docs/guides/image-generation).

## Privacy and licensing

Reference media never inherits the theme's CC BY 4.0 license. A package containing unresolved or non-redistributable references is marked `private_only` and must stay local. This public Release contains no APSAL-006 reference images or private Skill.

## Compatibility

- APSAL Open Protocol: 0.3.0
- Semantic Contract: 0.3.0
- Reference engine and Studio plugin: 0.5.0
- Existing 0.4 themes, runs, and Skills remain readable; they are not relabeled as reference-bound or native 4K.
