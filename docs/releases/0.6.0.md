# APSAL Studio 0.6.0

APSAL Studio 0.6 turns the local DNA Registry into an explainable creative memory and exchange system. APSAL recommends DNA from the requested scene, explains why it matches, asks before saving new knowledge to My DNA, learns from explicit outcomes, and exports reusable DNA independently from a theme.

## Install

```bash
codex plugin marketplace add henyjone/apsal-open --ref v0.6.0
codex plugin add apsal-studio@apsal-open
```

Restart Codex or open a new task. Then ask:

> Create a nine-shot Eastern-minimalist window portrait set. Explain each DNA recommendation and ask before saving new DNA to My DNA.

## Release assets

- `apsal-studio-codex-plugin-v0.6.0.zip` — self-contained Protocol 0.3 Registry, recommendation/memory engine, Extension Pack tools, MCP, Skill, schemas and 0.5 live-action/native-4K execution.
- `apsal-studio-codex-plugin-v0.6.0.zip.sha256` — SHA-256 checksum.

```bash
shasum -a 256 -c apsal-studio-codex-plugin-v0.6.0.zip.sha256
```

## Recommend and remember

For Character, World, Scene and Photo, APSAL ranks the compatible Registry subset using controlled tags/facets, explicit dependencies, rights, QA, scope and private outcome history. Every card states its reasons. Scope order is project → personal → extension → official, but scope never substitutes for scene fit or quality.

New or revised DNA receives proposed discovery metadata. After the creator confirms it, the project asset triggers exactly one choice: Save to My DNA, current project only, or decide later. APSAL never silently promotes DNA. Usage outcomes are local; the raw brief is not stored.

## Share DNA independently

A DNA Extension Pack contains canonical assets and preview sidecars rather than a full theme:

```text
apsal-dna-pack.json
checksums.sha256
LICENSE-CONTENT.md
README.md
registry/<namespace>/<type>/<id>/<version>/...
```

Public export requires one namespace, creator-confirmed tags, resolved rights/attribution, valid previews and dependencies. Private Vault media is never included. Install a pinned GitHub Release asset with:

```bash
apsal registry install 'github:owner/repo@v1.0.0#my-pack-v1.0.0.zip'
```

Installed packs are read-only and cannot override official or existing ID/version pairs.

## Compatibility and privacy

- APSAL Open Protocol: 0.3.0
- Semantic Contract: 0.3.0
- Reference engine and Studio plugin: 0.6.0
- DNA Extension Pack schema: 0.6.0
- Existing 0.4/0.5 themes, Registry assets, Skills and runs remain readable.
- Recommendation, memory, tagging, packaging and validation are local. A network call occurs only for an explicitly requested pinned GitHub Release install or image generation.
