# APSAL 十三元素：从摄影思想到语义契约

作者：“APSAL 创始人” HenyJone  
状态：Semantic Contract 0.3 试点章节  
内容许可：CC BY 4.0

## 阅读原则

协议字段保存机器可以验证的含义；本章解释这些字段为何具有摄影意义。经营位置、意境、气韵、虚实、三远与游观在这里是思想关联，不是古代概念与 JSON 字段的一对一翻译，也不是可直接量化的参数。

每个元素都按同一条路径解释：摄影与美学含义；中国视觉思想关联；世界模型作用；YAML 创作表达；canonical JSON；编译结果；正确与失败案例；QA、版本与权利。

## 十三个元素如何被创作者完整确认

十三个元素不是要求创作者面对十三份 JSON，也不等于建立十三个彼此割裂的 DNA 库。Studio 0.7 把它们组织为五次连贯对话：

| 创作层 | 协议元素 | 本层要回答的核心问题 |
|---|---|---|
| 命题与情绪 | Content、Emotion | 作品关于什么？总体是欢喜、悲伤、宁静、紧张，还是更复杂的混合？情绪怎样从开端走向转折与结尾？ |
| 人物与世界 | Subject、World、Look | 谁在世界中？空间、时间与物理规则是什么？服装、妆发、道具和所有权如何保持？ |
| 事件与序列 | Event、Sequence | 什么动作改变了世界？九次观看各自承担什么、留下什么后果？ |
| 摄影与成像 | Camera、Light、Style、Color/Post | 相机从哪里看？光从哪里来？摄影修辞、色温、饱和度、曲线、颗粒和肤色边界是什么？ |
| 执行与验证 | Job、Quality Control | 每个任务输出什么？什么结果必须拒绝？哪些判断必须由人看图确认？ |

这五层只是理解顺序，十三元素仍是协议，七类 DNA 仍是可复用资源。每一层先由 APSAL 根据自然语言与 Registry 提出方案，再由创作者接受或修改。只有十三元素全部拥有明确值、来源、可观察结果、不变量与 QA 标准，主题才可以冻结和执行。

## 1. Subject / 主体

1. **摄影与美学含义**：主体回答“谁存在”，身份不是脸部形容词的临时组合，而是跨视点仍可识别的稳定存在。
2. **中国视觉思想关联**：与“传神”和“气韵”形成关联，但机器不能只依赖抽象神似，仍需年龄、面部几何、发型和身体比例等明确锁。
3. **世界模型作用**：主体是身份连续性的最高优先级来源，情绪、动作和相机变化不得无声改写身份。
4. **YAML**：创作者声明 `purpose`、`must_preserve` 与允许变化的视线、表情、姿势。
5. **JSON**：规范化后保存稳定 ID、版本、依赖、权利及同一语义契约。
6. **编译**：设计目标解释身份为何必须固定；图像目标输出可观察身份特征；QA 目标生成跨镜比较项。
7. **案例**：《窗边未寄》九镜必须是同一虚构成年人物；脸型或年龄随景别变化属于失败。
8. **QA/版本/权利**：身份改变是 MAJOR；必须确认虚构或授权状态，个人身份锚点不得进入公开包。

## 2. World / 世界

1. **摄影与美学含义**：世界不是背景图，而是人物、空间、物件、时间和物理规则共同构成的持续现实。
2. **中国视觉思想关联**：关联意境、卧游以及“可行、可望、可游、可居”；图像应成为可以想象进入的世界。
3. **世界模型作用**：保存房间几何、方向、行动路径、物件归属和跨镜状态记忆。
4. **YAML**：明确窗户方向、桌子位置、可变的可见区域与不可变的空间关系。
5. **JSON**：成为依赖可解析、可摘要的 canonical world 模块。
6. **编译**：设计输出保留完整世界；单个图像 Prompt 只选择本镜可见部分，不要求把全部世界塞入画面。
7. **案例**：不同景别仍能推断为同一房间是正确；窗户数量和方向无故变化是漂移。
8. **QA/版本/权利**：世界前提改变使用 MAJOR；真实遗址、私人空间和参考媒体必须单独审查权利。

## 3. Style / 风格

1. **摄影与美学含义**：风格是一组稳定、可观察的视觉决定，不是“高级感”或艺术家姓名。
2. **中国视觉思想关联**：关联骨法和气韵，强调贯穿作品的组织方式，而不是表面装饰。
3. **世界模型作用**：协调材质、对比、颗粒、景深与负空间，但优先级低于身份、世界和事件。
4. **YAML**：说明风格目的、影响路径、可变强度及不可覆盖的材质真实。
5. **JSON**：使用可观察属性和负面边界，拒绝官方命名空间中的在世艺术家模仿映射。
6. **编译**：在 image target 的后段加入风格属性，冲突时不得压过身份和物理连续性。
7. **案例**：克制编辑摄影贯穿九镜是正确；每镜套用不同“热门风格”是失败。
8. **QA/版本/权利**：核心视觉方向改变通常为 MAJOR；参考风格必须记录来源与许可。

## 4. Look / 造型

1. **摄影与美学含义**：造型定义服装、妆发、饰品与道具如何属于人物、事件和时代。
2. **中国视觉思想关联**：关联“应物”与“随类”，形式应服从对象及情境。
3. **世界模型作用**：连接主体身份、事件动作、物件归属和序列连续性。
4. **YAML**：锁定 LOOK ID、道具所有者和状态，声明衣料运动或纸张方向等合法变化。
5. **JSON**：保存可复用 Look、依赖版本和精确状态路径。
6. **编译**：图像提示只描述当前状态；设计与 QA 输出保留所有权和继承规则。
7. **案例**：信纸由同一人物持有并随事件改变方向是正确；道具复制、漂浮或换手无因是失败。
8. **QA/版本/权利**：新增兼容造型用 MINOR；品牌、纹样、珠宝和服装设计需检查权利。

## 5. Emotion / 情绪

1. **摄影与美学含义**：情绪不是贴在脸上的标签，而是通过呼吸、视线、动作速度和空间距离显现的内在状态。
2. **中国视觉思想关联**：关联情景交融与气韵；情绪由人物和环境共同产生。
3. **世界模型作用**：把事件状态连接到人物外显行为和相机观看距离。
4. **YAML**：说明情绪目的、强度范围、允许变化和必须保持的克制边界。
5. **JSON**：保存可观察表现而非仅保存“悲伤”“高级”等抽象词。
6. **编译**：design target 解释内在阶段；image target 转译为凝视、呼吸和动作；QA 检查动机。
7. **案例**：决定通过停顿与翻纸被感知是正确；每镜相同模特式凝视是失败。
8. **QA/版本/权利**：情绪阶段扩充通常为 MINOR；不得从真实人物图像无依据推断敏感心理状态。

## 6. Event / 事件

1. **摄影与美学含义**：事件是改变世界状态的行为，区别于没有前因后果的姿势。
2. **中国视觉思想关联**：关联“生动”与“势”，动作应具有趋向和后果。
3. **世界模型作用**：产生 `state_before → action → state_after`，供下一镜继承。
4. **YAML**：说明事件目的、影响的物件和人物、不可颠倒的因果顺序。
5. **JSON**：记录稳定事件 ID、状态路径和依赖，不依赖自然语言顺序猜测。
6. **编译**：image target 输出可见动作；QA target 检查动作物理可读和状态后果。
7. **案例**：展开、触摸、翻面形成信纸状态链是正确；九镜只更换手势是失败。
8. **QA/版本/权利**：新增兼容事件用 MINOR；危险、违法或文化敏感事件需额外声明边界。

## 7. Camera / 相机

1. **摄影与美学含义**：相机回答谁在观看、从哪里观看、为什么此刻需要这一距离。
2. **中国视觉思想关联**：关联三远与游观，但不把一张拟真照片混合成无理由的多视点；原则是“一 Job 一视点，一序列一游观”。
3. **世界模型作用**：选择可见关系、画外空间、主体尺度和观看者的情绪距离。
4. **YAML**：为 framing、位置、高度、方向和镜头倾向解释用途。
5. **JSON**：保存单一视点和跨镜覆盖，平台相机参数仍属于适配层。
6. **编译**：design target保留视点理由；image target输出可观察取景；QA检查透视和几何。
7. **案例**：环境、全身、中景、近景、细节承担不同信息是正确；重复相同机位或破坏房间方向是失败。
8. **QA/版本/权利**：增加镜头覆盖用 MINOR；改变核心观看制度可能是 MAJOR。

## 8. Light / 光线

1. **摄影与美学含义**：光决定世界如何可见，也让时间、材质和事件阶段可以被感知。
2. **中国视觉思想关联**：关联虚实与明晦；这种关联用于解释观看，不把阴阳机械量化。
3. **世界模型作用**：连接窗户方向、时间阶段、阴影、肤色和序列节奏。
4. **YAML**：锁定来源与方向，允许色温、强度和衰减随阶段变化。
5. **JSON**：保存可验证的光源、阶段和负面规则。
6. **编译**：仅把当前 Job 的光阶段发送给图像模型；完整阶段关系保留在 design context。
7. **案例**：同一东窗从冷到暖是合法演化；多重太阳和矛盾阴影是失败。
8. **QA/版本/权利**：增加兼容阶段用 MINOR；静态验证不能证明生成光线已经正确。

## 9. Color Post / 色彩与后期

1. **摄影与美学含义**：色彩与后期组织对象、时间、材质和情绪，不是覆盖一切的滤镜。
2. **中国视觉思想关联**：关联“随类赋彩”，颜色服从对象与关系。
3. **世界模型作用**：保持肤色、空间材质和各光阶段之间的相对关系。
4. **YAML**：声明自然肤色、调色范围、颗粒与禁止的过度处理。
5. **JSON**：保存跨平台可观察结果，具体模型参数置于 adapter。
6. **编译**：在身份、事件、相机和光线之后加入成像属性。
7. **案例**：色彩随时间变暖但材质不漂移是正确；九镜随机滤镜化是失败。
8. **QA/版本/权利**：核心调色方向改变可能为 MAJOR；第三方 LUT 需单独许可。

## 10. Content / 内容

1. **摄影与美学含义**：内容回答作品真正关于什么，而不是画面里列出了哪些物体。
2. **中国视觉思想关联**：关联意与境；创作之“意”必须通过具体之“境”变得可感。
3. **世界模型作用**：为事件、情绪、序列和画外信息提供统一方向。
4. **YAML**：写世界命题、语义禁区和不依赖文字说明的表达目标。
5. **JSON**：保存主题命题、文化边界和内容许可。
6. **编译**：design target保留命题；image target不把抽象论述直接变成画面文字。
7. **案例**：“未说出口的决定”通过信纸事件表达是正确；生成字幕解释剧情是失败。
8. **QA/版本/权利**：世界命题变化通常为 MAJOR；文化遗产和敏感身份内容需专项审查。

## 11. Sequence / 序列

1. **摄影与美学含义**：序列让多次观看形成时间、节奏、信息分配和叙事。
2. **中国视觉思想关联**：关联卷轴、游观和步移景异；“散点透视”只作为现代概括谨慎使用。
3. **世界模型作用**：组织事件顺序、光线阶段、情绪强度和镜头覆盖。
4. **YAML**：说明每一 Job 的不可替代职能和前后状态。
5. **JSON**：保存顺序、依赖和状态继承，不把九张图当成无序数组。
6. **编译**：design target输出完整游观；image target逐 Job执行；QA检查重复与缺口。
7. **案例**：进入、接触、决定、余韵形成递进是正确；九张相似肖像是失败。
8. **QA/版本/权利**：新增兼容 Job 用 MINOR；移除关键叙事阶段可能改变生成意图。

## 12. Job / 镜头任务

1. **摄影与美学含义**：Job 是世界的一次完整观看，而不是九宫格中的一个格子。
2. **中国视觉思想关联**：关联经营位置，将共享世界重新组织成一个有目的的可见关系。
3. **世界模型作用**：把共享元素具体化为唯一视点、事件、状态和输出文件。
4. **YAML**：每镜记录 intent 与 framing、action、hands、gaze、composition 的字段意图。
5. **JSON**：生成唯一 Job ID、版本、依赖、文件名和摘要。
6. **编译**：分别产生设计语境、可观察图像 Prompt 和逐镜 QA。
7. **案例**：`SHOT_04` 的近景为了进入内在决定；只有 `close-up` 而没有目的属于弱设计。
8. **QA/版本/权利**：一 Job 一图、唯一文件名、无拼图文字水印；意图变化需记录版本。

## 13. Quality Control / 质量控制

1. **摄影与美学含义**：质量不是“看起来不错”，而是目标是否实现、非目标是否漂移以及证据是否充分。
2. **中国视觉思想关联**：六法提供整体品评传统，但现代 AI 摄影还需身份、解剖、反射、连续性、权利和披露检查。
3. **世界模型作用**：把每个元素的 `expected_effects` 转换成可接受或拒绝的证据条件。
4. **YAML**：创作时先写 QA expectation，而不是生成失败后再补负面词。
5. **JSON**：保存 QA ID、作用域、状态和证据链接。
6. **编译**：qa target生成中英双语全局与逐镜清单。
7. **案例**：Schema 通过只说明结构合法；没有生成图证据不得标记 `visual_qa_passed`。
8. **QA/版本/权利**：QA 规则增强通常为 MINOR；任何公开通过状态都必须可追溯。

## YAML、JSON 与专著的职责

```text
专著：解释为什么
YAML：表达本次创作为什么这样选择
JSON：保存规范事实
design：给 Codex 保留语义
image：给图像模型可观察语言
qa：把目的还原成证据检查
```

协议是机器规范，专著是思想与方法解释。专著发现的新问题通过 RFC、Schema、迁移记录和语义版本进入协议，不在文字中无声改变机器含义。
